# 第7章　Easy Musicとは何か

> 本章では、Easy Musicの設計思想、ACE-Step 1.5との関係性、システム構成、そして全機能の概要を紹介します。

---

## 7.1　Easy Musicの設計思想

Easy Musicは、「**ACE-Step 1.5を、誰でも簡単に、ブラウザだけで使えるようにする**」ことを目標に開発されたWebアプリケーションです。

ACE-Step 1.5は非常に強力な音楽生成モデルですが、REST APIやGradio UIを直接操作するには一定の知識が必要です。特に「良い楽曲を生成するためのプロンプト（キャプション・歌詞）をどう書くか」という問題は、初心者にとって大きなハードルとなります。

Easy Musicはこの問題を、以下の設計方針で解決します。

### ジャンルから始まるワークフロー

従来の「テキストプロンプトを自分で書く」アプローチではなく、**32種類のジャンルタイルをクリックする**ことから楽曲制作が始まります。ジャンルを選ぶだけで、テーマ生成→作詞→タグ生成→キャプション生成→音楽生成の全パイプラインが自動的に実行されます。

### AI作詞パイプライン

LLM（大規模言語モデル）を活用して、テーマから歌詞を自動生成します。ユーザーはジャンルを選ぶだけで、そのジャンルに適した物語とセクション構造（`[verse]`/`[chorus]`等）を持つ歌詞が自動的に作られます。さらに、歌詞からジャンル・楽器・BPM・調性を推定するAIタグ生成、ACE-Step 1.5に最適化されたキャプションを自動構成するAIキャプション生成が連鎖的に実行されます。

### 日本語ファースト

UIは日本語を第一言語として設計されており、180を超える翻訳キーによる完全な日英バイリンガル対応を実現しています。ジャンル名、ムード名、ボーカルタイプ名、エラーメッセージに至るまで、すべてのテキストが日本語で表示されます。ワンクリックで英語に切り替えることも可能です。

### シンプルな技術スタック

```
FastAPI（Python）+ バニラJavaScript（フレームワーク不要）
```

React、Vue、Next.jsなどのフロントエンドフレームワークは使用せず、素のHTML + CSS + JavaScriptで実装されています。ビルドツールは不要で、`pip install` と `python main.py` だけで起動できます。

## 7.2　Easy MusicとACE-Stepの関係

Easy Musicは、ACE-Step 1.5の**フロントエンド＋中間サーバー**として機能します。音楽生成エンジンそのものはACE-Step 1.5のAPIサーバーが担当し、Easy Musicはその上に付加価値を提供するレイヤーです。

```
┌──────────────────────────────────────────────────────────┐
│              Easy Music が提供する付加価値                 │
│                                                          │
│  ・32ジャンルのビジュアルグリッドUI                        │
│  ・AI作詞パイプライン（テーマ→歌詞→タグ→キャプション）    │
│  ・ムード/ボーカルのワンタッチ指定                        │
│  ・おまかせ生成（自然言語1行から全自動）                   │
│  ・JUKEBOXモード（連続自動生成・再生）                    │
│  ・カバー/リペイントモードUI                              │
│  ・フルスクリーンビジュアライザー（5種のモード）           │
│  ・生成履歴の管理                                        │
│  ・日英バイリンガルUI                                     │
│  ・デバッグパネル（キャプションの確認・手動編集）           │
└──────────────────────────────────────────────────────────┘
```

### ACE-Step 1.5が提供するもの

- DiT + LMによる音楽生成
- Text-to-Music / Cover / Repaint / Track Separation
- Format Input（プロンプトのAI強化、BPM・Key推定）
- ランダムサンプル生成
- 品質スコアリング / LRC生成

### Easy Musicが提供するもの

- ブラウザで操作可能なGUI
- LLMを活用した多段階の自動プロンプト構築
- ジャンル別に最適化されたキャプションヒント（32種）
- 外部LLM（Ollama等）+ ローカルLLM（内蔵Qwen3-1.7B）のフォールバック構成
- キャプション整形処理（ACE-Step 1.5の制約に準拠）
- 生成結果の無音検出と再試行促進

## 7.3　システム構成図

Easy Musicのシステムは、3つのレイヤーで構成されています。

```
┌─────────────────────────────────────────────────────────┐
│  ブラウザ（app.js + index.html + style.css）             │
│    ├ i18n（日英 180+キー）                               │
│    ├ ジャンルUI / おまかせ / Cover / Repaint / JUKEBOX    │
│    ├ ムード・ボーカルチップ                                │
│    ├ インラインプレーヤー + フルスクリーンビジュアライザー  │
│    ├ 生成履歴（localStorage）                             │
│    └ デバッグパネル                                       │
└──────────────┬──────────────────────────────────────────┘
               │ HTTP（fetch API）
┌──────────────┴──────────────────────────────────────────┐
│  Easy Music サーバー（FastAPI :8889）                     │
│    ├ routers/generate.py ── 音楽生成API                   │
│    ├ routers/lyrics.py ── LLM作詞API                     │
│    ├ services/ace_step_client.py ── ACE-Step通信          │
│    ├ services/llm_service.py ── 外部LLM通信               │
│    └ services/local_llm_service.py ── 内蔵Qwen3-1.7B     │
└──────────────┬──────────────────────────────────────────┘
               │ HTTP（httpx / openai）
       ┌───────┴──────┬──────────────────┐
       ▼              ▼                  ▼
┌──────────────┐ ┌──────────┐ ┌─────────────────┐
│ ACE-Step API │ │ 外部 LLM │ │ ローカル LLM    │
│  (:8001)     │ │ (Ollama  │ │ (Qwen3-1.7B     │
│              │ │  :11434) │ │  Q4_K_M.gguf)   │
│ DiT + LM    │ │ gemma3等 │ │ llama-cpp-python │
└──────────────┘ └──────────┘ └─────────────────┘
   音楽生成        作詞/テーマ      フォールバック
```

**通信プロトコル**: すべてHTTP/JSONです。ブラウザ↔Easy Music間はfetch API、Easy Music↔ACE-Step間はhttpx、Easy Music↔外部LLM間はOpenAI互換APIクライアントを使用します。

**LLMの3段構え**: Easy Musicは以下の優先順位でLLMを使用します。
1. **外部LLM**（Ollama等のOpenAI互換API）→ 最も高品質
2. **内蔵ローカルLLM**（Qwen3-1.7B-Q4_K_M.gguf）→ 外部LLM不可時のフォールバック
3. **ACE-Step内蔵LM**（Format Input機能）→ 最終手段

## 7.4　Easy Musicの全機能一覧

| 機能 | 説明 | 対応モード |
|------|------|-----------|
| **ジャンル選択（32種+ミックス）** | タイルUIで直感的にジャンルを選択。2ジャンルのブレンドも可能 | Normal |
| **AI テーマ生成** | ジャンルから物語・情景を自動生成（28ジャンル別コンテキスト） | Normal |
| **AI 作詞** | テーマ+ジャンル+ムード+ボーカルから構造化歌詞を生成 | Normal |
| **AI タグ生成** | 歌詞からジャンル/楽器/BPM/Keyを推定（JSON出力） | Normal |
| **AI キャプション生成** | ACE-Step向け最適化プロンプトを自動生成（2モード） | Normal |
| **おまかせ生成** | 自然言語1行から全自動生成（ACE-Step LMが全パラメータ決定） | Normal |
| **ランダム生成** | 🎲ボタンでパラメータをランダムセット | Normal |
| **カバーモード** | 音声ファイルのスタイル変換（強度0.00〜1.00） | Cover |
| **リペイントモード** | 音声の指定区間のみ再生成（秒単位で範囲指定） | Repaint |
| **JUKEBOXモード** | ジャンル選択→自動連続生成・再生（プリフェッチ付き） | JUKEBOX |
| **詳細設定** | Steps / Shift / Sampler / ネガティブプロンプト | 全モード |
| **ムードチップ（10種）** | ワンタッチで雰囲気を指定 | Normal |
| **ボーカルチップ（6種）** | ワンタッチで歌声タイプを指定 | Normal |
| **インラインプレーヤー** | 複数トラック切り替え再生・シークバー・音量制御 | 全モード |
| **フルスクリーンビジュアライザー** | 5種のビジュアルモード + ジャンル別背景 | 全モード |
| **生成履歴** | 直近の生成結果を保持（localStorage永続化） | 全モード |
| **デバッグパネル** | 生成パラメータの確認・キャプション手動編集 | 全モード |
| **無音検出** | 生成結果の無音を自動判定し再試行を促す | 全モード |
| **日英バイリンガルUI** | ワンクリックで言語切替（180+キーの完全翻訳） | 全モード |

### モード排他制御

Easy Musicは4つのモードを持ちますが、**同時に1つのモードのみ**がアクティブになります。

| モード | トリガー | 他モードへの影響 |
|--------|---------|----------------|
| **Normal** | デフォルト | — |
| **Cover** | 🎤ボタン | Normal表示を隠し、Cover専用パネルを表示 |
| **Repaint** | 🎨ボタン | Normal表示を隠し、Repaint専用パネルを表示 |
| **JUKEBOX** | 🎶ボタン | ジャンルを複数選択モードに切替 |

JUKEBOX実行中（曲を連続生成・再生中）は、Cover/Repaintへの切り替えがブロックされます。

## コラム: なぜバニラJavaScriptなのか

Easy Musicは、React、Vue、Svelteなどのモダンフレームワークを一切使用せず、素のJavaScript（3,074行の単一ファイル `app.js`）で実装されています。

この選択には以下の理由があります。

1. **ビルドステップ不要**: `npm install`、`webpack`、`vite` などのビルドツールが不要。`pip install` だけで動作します
2. **依存関係の最小化**: Node.js環境を必要としないため、Python環境のみで動作します。ACE-Step 1.5自体がPython環境を前提としているため、追加の環境構築が不要です
3. **透明性**: ソースコードがそのまま実行されるため、ブラウザのDevToolsで直接デバッグ可能です
4. **学習容易性**: フレームワーク固有の概念（仮想DOM、コンポーネントライフサイクル、状態管理ライブラリ等）の知識が不要です

トレードオフとして、大規模な機能追加ではコードの管理が難しくなりますが、Easy Musicの規模（HTML 525行 + JS 3,074行 + CSS）では十分に管理可能です。

## 8.2　ACE-Step APIサーバーの準備

Easy Musicを使用するには、まずACE-Step 1.5のAPIサーバーを起動する必要があります。

### Step 1: uvのインストール

```bash
# Linux / macOS
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### Step 2: ACE-Step 1.5のクローンとインストール

```bash
git clone https://github.com/ACE-Step/ACE-Step-1.5.git
cd ACE-Step-1.5
uv sync
```

### Step 3: APIサーバーの起動

```bash
# REST APIサーバーを起動
uv run acestep-api
# → http://localhost:8001 でAPIサーバーが起動
```

初回起動時にモデルファイル（合計約10.1GB）がHugging Faceから自動ダウンロードされます。回線速度によっては数分〜数十分かかります。

### GPU別モデル自動選択

ACE-Step 1.5は、GPUのVRAM容量を自動検出し、最適なモデル構成を選択します。

| VRAM | DiTモデル | LMモデル | バックエンド |
|------|----------|---------|------------|
| ≤ 6GB | turbo (INT8量子化) | なし | — |
| 6〜8GB | turbo | 0.6B | PyTorch |
| 8〜16GB | turbo | 0.6B〜1.7B | vLLM |
| 16〜24GB | turbo | 1.7B | vLLM |
| ≥ 24GB | turbo | 4B | vLLM |

手動で変更したい場合は、`.env` ファイルで指定できます。

```bash
# ACE-Step-1.5/.env
ACESTEP_CONFIG_PATH=acestep-v15-turbo    # DiTモデル
ACESTEP_LM_MODEL_PATH=acestep-5Hz-lm-1.7B  # LMモデル
```

### 起動の確認

```bash
curl http://localhost:8001/health
# → {"status": "ok"}
```

このレスポンスが返ればACE-Step APIサーバーは正常に動作しています。

## 8.6　LLM構成の選択

Easy Musicは、テーマ生成・作詞・タグ生成・キャプション生成にLLMを使用します。3つの構成から選択できます。

### 構成1: 外部LLM（推奨）

| 項目 | 詳細 |
|------|------|
| **方式** | Ollama / vLLM / OpenAI互換API |
| **推奨モデル** | gemma3:latest、qwen3:4b、GPT-4o mini |
| **利点** | 高品質な歌詞・テーマ生成、多言語対応 |
| **欠点** | 別途サーバー（Ollama等）の起動が必要 |
| **設定** | `.env` でURL/モデルを指定 |

```bash
# Ollamaの場合
ollama serve                        # Ollamaサーバー起動
ollama pull gemma3:latest           # モデルダウンロード
python main.py                      # デフォルトでOllamaに接続
```

### 構成2: 内蔵ローカルLLM

| 項目 | 詳細 |
|------|------|
| **方式** | llama-cpp-python + Qwen3-1.7B-Q4_K_M.gguf |
| **利点** | 追加サーバー不要、自動ダウンロード |
| **欠点** | 品質は外部LLMに劣る、CPU推論のため低速 |
| **設定** | `--local-llm` フラグ、または `.env` で `USE_LOCAL_LLM_ONLY=true` |

内蔵LLMは、外部LLMへの接続が確立できなかった場合の**自動フォールバック**としても機能します。モデルファイル（約1.1GB）は初回使用時にHugging Face（`unsloth/Qwen3-1.7B-GGUF`）から自動ダウンロードされ、`models/` ディレクトリに保存されます。

### 構成3: LLMなし（ACE-Step内蔵LMのみ）

| 項目 | 詳細 |
|------|------|
| **方式** | テーマ・作詞機能を使わず、おまかせ生成/ランダム生成のみ使用 |
| **利点** | Easy Musicサーバーのみで動作。最も軽量 |
| **欠点** | AI作詞・テーマ・タグ・キャプション生成が使用不可 |
| **設定** | 特別な設定は不要。LLM接続エラーは無視される |

この場合、ACE-Step 1.5内蔵のLM（Format Input / Random Sample機能）がプロンプト生成を担当します。

### フォールバックチェーン

```
テーマ/歌詞/タグ/キャプション生成リクエスト
         │
         ▼
  ┌──────────────┐
  │ 外部LLM       │ ─── 成功 → 結果を返す
  │ (Ollama等)    │
  └──────┬───────┘
         │ 失敗
         ▼
  ┌──────────────┐
  │ ローカルLLM    │ ─── 成功 → 結果を返す
  │ (Qwen3-1.7B) │
  └──────┬───────┘
         │ 失敗
         ▼
  ┌──────────────┐
  │ エラー返却     │ ─── フロントエンドに通知
  └──────────────┘
```

## 8.7　起動確認チェックリスト

すべてのコンポーネントが正常に起動しているか確認するためのチェックリストです。

| # | 確認項目 | 確認方法 | 期待結果 |
|---|---------|---------|---------|
| 1 | ACE-Step APIサーバー | `curl http://localhost:8001/health` | `{"status": "ok"}` |
| 2 | Easy Musicサーバー | ブラウザで `http://localhost:8889` | Easy MusicのUIが表示される |
| 3 | ACE-Step接続 | Easy Musicで `🎵 音楽を生成` をクリック | エラーなく生成が開始される |
| 4 | 外部LLM（任意） | Easy Musicで `/api/llm_status` にアクセス | `external_llm: true` |
| 5 | モデル情報 | Easy Musicの UIでモデル名が表示される | "Turbo(高速)" 等が選択可能 |

### トラブルシューティング

| 症状 | 原因 | 対処法 |
|------|------|--------|
| Easy Music起動時にエラー | 依存関係未インストール | `pip install -r requirements.txt` |
| 「ACE-Step APIに接続できません」 | APIサーバー未起動 or URL不一致 | ACE-Step APIの起動を確認、`--ace-url` を確認 |
| テーマ/歌詞が生成されない | LLM未接続 | Ollama起動を確認、または `--local-llm` で内蔵LLMを使用 |
| モデルダウンロードが遅い | ネットワーク速度 | 待機（合計約10.1GB + 1.1GB） |
| ポートが使用中 | 他のプロセスが占有 | `--port 9000` で別ポートを指定 |

# 第9章　チュートリアル — はじめての音楽生成

> 本章では、Easy Musicの基本操作を7つのチュートリアルで実践的に学びます。ジャンル選択からの1曲生成、カバー、リペイント、JUKEBOXモードまで、画面操作を追いながら体験しましょう。

**前提条件**: 第8章の手順でACE-Step 1.5 APIサーバーとEasy Musicが起動済みであること。ブラウザで `http://localhost:8889` にアクセスできる状態で始めてください。

## 9.1　チュートリアル1: ジャンル選択から1曲生成する

最もシンプルな操作から始めましょう。ジャンルを1つ選んでボタンを押すだけで、AIが自動的にテーマ考案→作詞→キャプション生成→音楽生成のパイプラインを実行します。

### Step 1: ジャンルを選択する

画面上部のジャンルグリッドから **「J-POP」** タイルをクリックします。タイルが青く光り、選択状態になります。

> 💡 ジャンルタイルには「キャプションヒント」が埋め込まれています。J-POPの場合:
> `J-POP, Japanese pop, catchy melody, bright, emotional vocal, polished production`
> このヒントがAI生成の方向性を決定します。

### Step 2: 「🎵 音楽を生成」ボタンをクリック

画面下部の生成ボタンをクリックします。ボタンがグレーアウトし、プログレスバーが表示されます。

### Step 3: 自動パイプラインの動作を観察する

ボタンの下に進捗メッセージが表示されます。内部では以下の処理が順に実行されています:

```
① テーマ生成（/api/theme）
   → LLMがJ-POPに合ったテーマを自動考案
   例: 「桜舞い散る春の午後、新しい出発を決意した少女の物語」

② 歌詞生成（/api/lyrics）
   → テーマから[verse][chorus]構造の歌詞を自動生成

③ キャプション生成（/api/caption）
   → 歌詞とテーマからACE-Step用の音楽記述文を生成

④ AI強化（/api/format_input）
   → ACE-StepのLMがBPM・Key・詳細キャプションを補完

⑤ 音楽生成（/api/generate）
   → DiTモデルが楽曲を生成（Turboモデルで約15〜30秒）
```

### Step 4: 生成結果を再生する

生成完了後、インラインプレーヤーが表示されます。再生ボタン（▶）をクリックして楽曲を聴きましょう。

- **トラック切替**: バッチ生成時は `◀ ▶` ボタンでトラックを切り替えられます
- **シークバー**: クリックまたはドラッグで再生位置を変更できます
- **音量**: 音量スライダーで調整できます

### Step 5: フルスクリーンビジュアライザーで鑑賞する

プレーヤーエリアをクリックすると、フルスクリーンビジュアライザーが表示されます。

- **5つのビジュアルモード**: Spectrum / Wave / Ring / Particles / Pulse
- 画面タップでモードが切り替わります
- 歌詞がリアルタイムで表示されます
- もう一度タップするとビジュアライザーを閉じます

> 🎉 これが最もシンプルな使い方です。ジャンルを選んでボタンを押すだけで、AIが全工程を自動処理します。

## 9.2　チュートリアル2: テーマと歌詞を指定して生成する

AIに任せきりではなく、テーマや歌詞の方向性を自分で決めて生成してみましょう。

### Step 1: ジャンルとムード・ボーカルを選択する

1. ジャンルグリッドから **「アニソン」** を選択
2. ムードチップから **「✨ 神秘的」** を選択（紫色にハイライト）
3. ボーカルチップから **「👩 女声」** を選択

### Step 2: 作詞設定を開く

画面下部の **「✍ 作詞設定」** アコーディオンをクリックして展開します。

### Step 3: テーマを入力する

テーマ入力欄に以下のように入力します:

```
月夜の森で迷う少女が、光る蝶に導かれて星の泉にたどり着く物語
```

具体的な情景・キャラクター・展開を含めると、より高品質な歌詞が生成されます。

### Step 4: 歌詞を入力する（任意）

歌詞入力欄に自分で書いた歌詞を入力することもできます。セクションタグを使った構造で書きます:

```
[verse]
月明かりが照らす 深い森の奥
迷い込んだ少女は 一人立ち止まる
光る蝶が舞い降りて 道を示すように
不思議な輝きに 心が惹かれてゆく

[chorus]
星の泉よ 導いて
この暗闇を抜けて 光の先へ
翼のない私でも 飛べると信じて
月夜に響く 希望の歌
```

> 💡 歌詞を空欄にしておけば、テーマからAIが自動で作詞します。サビだけ書いて残りをAIに任せることも可能です。

### Step 5: 生成して結果を確認する

「🎵 音楽を生成」をクリック。テーマとムード・ボーカルの指定が反映された楽曲が生成されます。

画面下部の **デバッグパネル** （折りたたみ）を開くと、実際に使用されたキャプション、BPM、Key、タグなどの詳細を確認できます。

## 9.3　チュートリアル3: おまかせ生成を使う

「おまかせ生成」は、自然言語で1行のリクエストを入力するだけで、ACE-Step 1.5内蔵のLMがジャンル・BPM・Key・歌詞・キャプションをすべて自動決定する機能です。

### Step 1: おまかせフィールドに入力する

ジャンルグリッドの下にある **「おまかせ生成」** 入力欄に、作りたい曲のイメージを1行で入力します:

```
夏の海辺で聴きたい爽やかなポップ
```

### Step 2: 生成する

「🎵 音楽を生成」をクリックすると、通常のパイプラインではなくACE-Stepの `/format_input` APIが呼ばれ、LMがすべてのパラメータを自動決定します。

> 💡 おまかせ生成では、Easy Music側のLLM（Ollama等）は使用されません。ACE-Step 1.5内蔵のLM（Qwen3-Embedding-0.6B等）が直接処理します。

### Step 3: ランダム生成を試す

おまかせ入力欄の右にある **🎲（サイコロ）** ボタンをクリックすると、ACE-Stepの `/create_random_sample` APIからランダムなパラメータセットが取得され、UIに自動入力されます。

ランダム結果をベースに、言語やジャンルだけ変更して生成するのも楽しみ方の一つです。

### 効果的なおまかせ入力の例

| 入力例 | 説明 |
|-------|------|
| 夏の海辺で聴きたい爽やかなポップ | 季節感＋場所＋雰囲気 |
| 深夜のバーで流れるようなジャズバラード | 場所＋時間帯＋ジャンル |
| アニメのオープニングテーマ風の激しいロック | メディア参照＋ジャンル |
| 雨の日に部屋で聴くローファイビート | 天気＋シチュエーション＋ジャンル |
| A dreamy synthwave track for a neon-lit city drive | 英語でもOK |

## 9.4　チュートリアル4: カバーモードで既存曲をアレンジする

カバーモードは、既存の音声ファイルをベースに、AIがスタイルを変換した新しい楽曲を生成する機能です。

### Step 1: カバーモードに切り替える

画面上部のヘッダーにある **「🎤 カバー」** ボタンをクリックします。通常のジャンルグリッドやコントロールが非表示になり、カバー専用パネルが表示されます。

> ⚠ カバーモードと通常モード、リペイントモード、JUKEBOXモードは排他的です。カバーをONにすると他のモードは自動的にOFFになります。

### Step 2: 音声ファイルをアップロードする

カバーパネルのアップロードエリアに音声ファイルをドラッグ＆ドロップします。クリックしてファイル選択ダイアログから選ぶこともできます。

対応フォーマット: MP3、WAV、FLAC、OGG

### Step 3: カバー強度を調整する

**カバー強度スライダー**（Audio Cover Strength）で変換の度合いを設定します:

| 値 | 効果 | 用途 |
|-----|------|------|
| 0.00 | 原曲そのまま | テスト用 |
| 0.30 | 軽微な変化 | 微調整 |
| **0.50** | **中程度の変換（推奨）** | **原曲の骨格を残しつつ変化** |
| 0.80 | 大幅な変換（デフォルト） | ジャンル転換 |
| 1.00 | 完全に新しい解釈 | 実験的 |

> 💡 初めて使う場合は **0.50** がおすすめです。原曲のメロディやリズムの骨格が残りつつ、新しいアレンジが加わります。

### Step 4: キャプションを入力する

キャプション入力欄に、変換後のスタイルを英語で指定します:

```
jazz, acoustic, warm piano, smooth bass, relaxed tempo
```

### Step 5: 生成して原曲との変化を確認する

「🎵 音楽を生成」をクリック。生成はmultipart形式でACE-Stepに送信されます。

完成したカバー楽曲を再生し、原曲と聴き比べてみましょう。カバー強度やキャプションを変えて複数パターンを試すことで、最適なアレンジを見つけられます。

> 💡 カバーモードでは `guidance_scale` が自動的に `1.0` 以上に補正されます。これはACE-Step 1.5で カバー生成時に低い値だと品質が低下するためです。

## 9.5　チュートリアル5: リペイントモードで部分修正する

リペイントモードは、生成済みの楽曲の一部区間だけを指定して再生成する機能です。「サビだけジャンルを変えたい」「イントロを延長したい」といった部分的な編集に最適です。

### Step 1: リペイントモードに切り替える

ヘッダーの **「🎨 リペイント」** ボタンをクリックします。リペイント専用パネルが表示されます。

### Step 2: 音声ファイルをアップロードする

先ほど生成した楽曲（またはチュートリアル4のカバー結果）をダウンロードし、リペイントパネルにアップロードします。

### Step 3: 再生成区間を設定する

2つの数値入力フィールドで再生成する区間を秒単位で指定します:

- **開始秒（Repainting Start）**: `15.0`
- **終了秒（Repainting End）**: `30.0`

この設定では、15秒目から30秒目の15秒間だけが再生成され、0〜15秒と30秒以降はそのまま保持されます。

### Step 4: キャプションでスタイルを指定する

再生成部分のスタイルをキャプションで指定します:

```
epic orchestral, dramatic strings, powerful drums, cinematic climax
```

### Step 5: 生成して結果を確認する

「🎵 音楽を生成」をクリック。指定区間のみが新しいスタイルで再生成された楽曲が得られます。

再生して、区間の境界がどの程度自然につながっているかを確認しましょう。ACE-Step 1.5のリペイント機能は遷移部分の滑らかさをある程度自動的に処理しますが、極端なスタイル変更では不自然な接続になることもあります。

> 💡 リペイントを繰り返し適用することで、セクションごとに異なるスタイルの楽曲を構成することも可能です。ただし、反復適用による品質低下に注意してください。

## 9.6　チュートリアル6: JUKEBOXモードで連続再生する

JUKEBOXモードは、AIが自動的に曲を次々と生成・再生し続ける、BGM自動生成マシンです。

### Step 1: JUKEBOXモードに切り替える

ヘッダーの **「🎶 自動演奏」** ボタンをクリック。画面がJUKEBOX専用のUIに切り替わります。

### Step 2: ジャンルを複数選択する

JUKEBOXモードでは、ジャンルタイルを複数選択できます。選択したジャンルからランダムに1つが選ばれて、曲が生成されます。

おすすめの組み合わせ例:
- **カフェBGM**: Jazz + Bossa Nova + Lo-Fi + Acoustic Pop
- **ドライブミュージック**: Rock + City Pop + Driving + EDM
- **リラックス**: Folk + 80年代フォーク + Lo-Fi + Mood
- **アニメソング集**: アニソン + J-POP + ボカロ + Rock

### Step 3: 「▶ スタート」をクリックする

自動生成が始まります。内部では以下のループが動作します:

```
┌─────────────────────────────────────────┐
│  ① 選択ジャンルからランダムに1つ選択     │
│  ② テーマ自動生成                        │
│  ③ 歌詞生成                             │
│  ④ キャプション生成                      │
│  ⑤ 音楽生成（guidance_scale=18.0）       │
│  ⑥ 再生開始                             │
│  ⑦ 再生中に次の曲をプリフェッチ          │
│  ⑧ 現在の曲が終わったら次の曲をシームレスに再生 │
│  └──→ ①に戻る（無限ループ）             │
└─────────────────────────────────────────┘
```

> 💡 JUKEBOXモードでは `guidance_scale` が `18.0` に設定されます。これは多様なジャンルで安定した品質を得るための設定です。

### Step 4: ビジュアライザーを楽しむ

再生中にプレーヤーエリアをタップすると、フルスクリーンビジュアライザーが起動します。ビジュアルモードは曲ごとにランダムに切り替わります。

### Step 5: 停止する

**「⏹ ストップ」** をクリックすると、現在再生中の曲が終わった時点で停止します。即座に再生を止めたい場合はプレーヤーの停止ボタンを使います。

### 自動リトライ機能

JUKEBOXモードでは、生成に失敗した場合に自動的にリトライします（最大3回）。3回連続で失敗した場合は自動的に停止し、エラーメッセージが表示されます。

## 9.7　チュートリアル7: 詳細設定を使いこなす

Easy Musicの詳細設定を使って、生成品質やスタイルを細かくコントロールしてみましょう。

### 詳細設定パネルを開く

**「⚙ 詳細設定」** アコーディオンをクリックして展開します。

### Steps（推論ステップ数）

DiTモデルの推論（ノイズ除去）回数を制御します。

| 値 | 生成時間 | 品質 | 用途 |
|-----|---------|------|------|
| **8** | 約15〜30秒 | 十分実用的 | Turboモデルのデフォルト。素早いプロトタイプに |
| 20 | 約30〜60秒 | 良好 | バランスの取れた選択肢 |
| 50 | 約1〜3分 | 高品質 | 仕上げ段階での使用に |
| 100 | 約3〜8分 | 最高品質 | 最終作品の生成に |

> 💡 Turboモデル（DMD蒸留済み）は8ステップで十分な品質が出ます。Baseモデルを使う場合は50ステップ以上がおすすめです。

### Shift（タイムステップシフト）

ノイズ除去プロセスの焦点を制御するパラメータです。

| 値 | 効果 | 説明 |
|-----|------|------|
| auto | 自動最適化 | モデルに任せる（推奨） |
| 1.0 | ディテール重視 | 細かい音色やニュアンスに注力 |
| 2.0〜3.0 | バランス型 | 構造とディテールの中間 |
| 5.0 | 構造重視 | 楽曲全体の構成に注力 |

### Sampler（サンプリング方式）

ノイズ除去の数学的アルゴリズムを選択します。

| 方式 | 特徴 | 向いている用途 |
|------|------|--------------|
| **auto (ODE)** | 安定した生成、再現性が高い | 汎用、デフォルト |
| ODE | 常微分方程式ベース | 安定した結果が欲しい場合 |
| SDE | 確率微分方程式ベース | より有機的・自然な音が欲しい場合 |

> 💡 SDEは同じパラメータでも毎回異なる結果が出やすく、「人間味」のある音になる傾向があります。ただしODEに比べてステップ数を多めに設定する必要があります。

### キャプション自動/Thinking

| 設定 | 動作 |
|------|------|
| **キャプション自動 ON** | ACE-StepのLMがキャプションを強化（推奨） |
| キャプション自動 OFF | Easy MusicのLLMが生成したキャプションをそのまま使用 |
| **Thinking ON** | LLMが思考プロセス付きで簡潔な1文キャプションを生成 |
| Thinking OFF | LLMが3〜5文の詳細なキャプションを直接生成 |

### ネガティブプロンプト

「こうならないでほしい」という指示を記述します。デフォルト値:

```
low quality, noisy, distorted, muddy, clipping, off-key, out of tune, amateur, poorly mixed
```

効果的なカスタム例:
- ピアノを避けたい場合: `piano, keyboard`
- 激しい曲を避けたい場合: `aggressive, heavy, distorted guitar`
- 電子音を避けたい場合: `synth, electronic, digital`

### バッチ生成

曲数を **2** または **4** に設定すると、同じパラメータで複数の楽曲が同時に生成されます。同じ設定でもseed値が異なるため、それぞれ異なる楽曲が生まれます。

**使い方のコツ**: まず4曲同時生成し、トラック切替ボタン（◀ ▶）で聴き比べて、最も気に入った曲をダウンロードする。プロの音楽制作でも「複数テイクから最良を選ぶ」のは一般的な手法です。

## まとめ: 7つのチュートリアルで学んだこと

| チュートリアル | 学んだこと | キーポイント |
|-------------|---------|------------|
| 1: ジャンル選択から生成 | 基本操作 | ジャンル選択→ボタン1クリックで完成 |
| 2: テーマと歌詞指定 | カスタマイズ | ムード・ボーカル・テーマで方向性制御 |
| 3: おまかせ生成 | AI任せ | 自然言語1行で全自動、🎲でランダム |
| 4: カバーモード | 既存曲の変換 | 強度スライダーで変換度合いを調整 |

| チップ | コード値 | 説明 |
|--------|---------|------|
| 😊 明るい | `bright and happy` | ポジティブ、明るい雰囲気 |
| 😢 切ない | `bittersweet and sad` | 切なさ、哀愁 |

| 💕 ロマンチック | `romantic and tender` | ロマンティック、優しい |
| 🎉 楽しい | `playful and fun` | 楽しい、陽気 |

ボーカルスタイルを指定します。キャプションに声の特徴が追加されます。

| チップ | コード値 | 説明 |
|--------|---------|------|
| 👩 女声 | `female vocal` | 女性ボーカル |

### テーマ入力

テーマ欄にはAIに作曲の方向性を指示するテキストを入力します。

- **空欄**: LLMがジャンルに合ったテーマを自動生成します
- **入力あり**: 入力されたテーマに基づいて歌詞が生成されます
- **プレースホルダー例**: 「夜空を見上げて願いを込める魔法少女の物語」

### 歌詞入力

歌詞欄にはセクションタグ付きの歌詞テキストを入力します。

- **空欄**: テーマからAIが自動で作詞します
- **入力あり**: 入力された歌詞がそのまま使用されます

対応セクションタグ:
```
[verse]    — Aメロ・Bメロ
[chorus]   — サビ
[bridge]   — ブリッジ、大サビ
[intro]    — イントロ
[outro]    — アウトロ
[pre-chorus] — プレコーラス
[interlude]  — 間奏
```

### 歌詞再生成ボタン（♻️）

歌詞入力欄の横にある **♻️ 再生成** ボタンをクリックすると、現在のテーマとジャンル設定に基づいて歌詞をLLMで再生成します。

### ネガティブプロンプト

生成時に避けたい要素を指定します。

- **空欄時のデフォルト値**: `low quality, noisy, distorted, muddy, clipping, off-key, out of tune, amateur, poorly mixed`
- **カスタム入力**: 避けたい楽器やスタイルを自由に記述

## 10.7　おまかせ生成

**「🪄 おまかせ生成」** セクションは、自然言語で1行のリクエストを入力するだけで全パラメータを自動決定する機能です。

### 動作の仕組み

1. 入力テキストをACE-Stepの `/format_input` APIに送信
2. ACE-Step内蔵のLM（Qwen3-Embedding等）がジャンル・BPM・Key・歌詞・キャプションをすべて推定
3. 推定結果に基づいて音楽を生成

> 💡 おまかせ生成ではEasy Music側のLLM（Ollama等）は使用されません。ACE-Stepの内蔵LMが直接処理します。

### 🎲 ランダムボタン

ACE-Stepの `/create_random_sample` APIからランダムなパラメータセットを取得し、UIに自動入力します。取得されるパラメータには、ジャンル、BPM、Key、歌詞、キャプションが含まれます。

言語ドロップダウンで選択中の言語は保持されます。

## 10.8　カバーモード

ヘッダーの **「🎤 カバー」** ボタンをクリックするとカバーモードに切り替わります。

### カバーパネルの構成

| 要素 | 説明 |
|------|------|
| **アップロードエリア** | 音声ファイルのドラッグ＆ドロップまたはクリック選択。`accept="audio/*"` |
| **ファイル情報表示** | ファイル名と❌クリアボタン |
| **カバー強度スライダー** | 0.00〜1.00。デフォルト: 0.80 |
| **キャプション入力** | 変換後のスタイル指定（例: jazz, acoustic, lo-fi） |
| **歌詞入力（折りたたみ）** | 任意。新しい歌詞を入力可能 |

### カバー強度の詳細

| 範囲 | 効果 |
|------|------|
| 0.00〜0.20 | 原曲のほぼそのまま。テスト用 |
| 0.20〜0.50 | 軽微な変化。原曲の骨格が明確に残る |
| 0.50〜0.70 | 中程度の変換。メロディの輪郭は残るがアレンジが変化 |
| 0.70〜0.90 | 大幅な変換。ジャンル転換に最適 |
| 0.90〜1.00 | 完全に新しい解釈。原曲との関連が薄い |

> ⚠ カバーモードでは `guidance_scale` が自動的に `1.0` 以上に補正されます。ACE-Step 1.5でカバー生成時に低い `guidance_scale` は品質低下の原因となるためです。

## 10.9　リペイントモード

ヘッダーの **「🎨 リペイント」** ボタンをクリックするとリペイントモードに切り替わります。

### リペイントパネルの構成

| 要素 | 説明 |
|------|------|
| **アップロードエリア** | 音声ファイルのドラッグ＆ドロップまたはクリック選択 |
| **再生成区間（秒）** | 開始秒と終了秒を数値入力。step=0.1で0.1秒単位 |
| **キャプション入力** | 再生成部分のスタイル指定（任意） |
| **歌詞入力（折りたたみ）** | 再生成部分の歌詞（任意） |

### 再生成区間の仕様

- **開始秒（Repainting Start）**: 再生成を開始する位置（秒）
- **終了秒（Repainting End）**: 再生成を終了する位置（秒）
- **バリデーション**: 終了秒 > 開始秒 でなければエラー
- 指定区間外の音声はそのまま保持されます

## 10.10　JUKEBOXモード

### JUKEBOX専用UI

| 要素 | 説明 |
|------|------|
| **ジャンルグリッド（複数選択）** | タイルを複数タップして選択。選択されたジャンルは青くハイライト |
| **選択カテゴリバッジ** | 選択中のジャンルをバッジで表示 |
| **Now Playing表示** | 現在再生中の曲の情報を表示 |
| **▶ スタート / ⏹ ストップ** | 自動生成ループの開始/停止 |
| **シークバー / 音量** | JUKEBOX専用のプレーヤーコントロール |

### 動作フロー

```
1. 選択ジャンルからランダムに1つを選出
2. テーマ自動生成（/api/theme）
3. 歌詞生成（/api/lyrics）
4. キャプション生成（/api/caption）
5. 音楽生成（/api/generate, guidance_scale=18.0）
6. 再生開始
7. 再生中に次の曲をプリフェッチ（バックグラウンドで生成開始）
8. 現在の曲が終了 → プリフェッチ済みの次の曲をシームレスに再生
9. → ステップ1に戻る（無限ループ）
```

### 停止動作

- **⏹ ストップ**: 現在再生中の曲が終わった時点で停止
- 「🛑 現在の曲の再生後に停止します...」というメッセージが表示されます

### 自動リトライ

- 生成失敗時は自動でリトライ（最大3回）
- 3回連続失敗で「❌ 生成に3回失敗しました。JUKEBOXを停止します。」と表示して停止

## 10.11　インラインプレーヤーとビジュアライザー

### インラインプレーヤー

楽曲生成完了後に表示されるプレーヤーコントロール。

| 要素 | 説明 |
|------|------|
| **◀ / ▶ トラック切替** | バッチ生成時に複数トラックを切り替え |
| **再生/一時停止ボタン** | 楽曲の再生制御 |
| **シークバー** | クリックまたはドラッグで再生位置を変更 |
| **時間表示** | 現在再生位置 / 総時間 |
| **音量スライダー** | 音量調整 |
| **ダウンロードボタン** | 楽曲をMP3形式でダウンロード |

### フルスクリーンビジュアライザー

プレーヤーエリアをタップするとフルスクリーンのビジュアライザーオーバーレイが表示されます。

#### 5つのビジュアルモード

| モード | 説明 | 技術 |
|--------|------|------|
| **Spectrum** | FFTデータのバー表示。周波数スペクトルを視覚化 | AnalyserNode + getByteFrequencyData |
| **Wave** | 波形データのカーブ表示。リアルタイム波形 | AnalyserNode + getByteTimeDomainData |
| **Ring** | 円環パターンのスペクトル。円状に配置されたバー | 極座標変換 + FFTデータ |
| **Particles** | 音声駆動パーティクルシステム。音に反応する粒子 | パーティクル生成 + 物理シミュレーション |
| **Pulse** | 音量連動パルスエフェクト。ビートに合わせて脈動 | RMS計算 + スケーリング |

#### ビジュアライザーの技術基盤

- **Web Audio API**: `AudioContext` + `AnalyserNode`
- **Canvas 2D**: `requestAnimationFrame` によるリアルタイムレンダリング
- **ジャンル別背景画像**: 各ジャンルに対応した背景が表示される
- **歌詞のリアルタイム表示**: 生成時に使用された歌詞をオーバーレイ表示

画面タップでビジュアルモードが切り替わり、もう一度タップするとオーバーレイを閉じます。

## 10.13　デバッグパネル

**「詳細情報」** ボタンで折りたたみを展開すると、最後の生成で使用されたパラメータの詳細が確認できます。

| 表示項目 | 説明 |
|---------|------|
| **ジャンル** | 選択されたジャンル名 |
| **テーマ** | 使用されたテーマテキスト |
| **モード** | 使用されたモード（キャプション自動/外部LLM/手動等） |
| **タグ (raw)** | LLMが生成したタグ情報（JSON） |
| **キャプション** | 最終的にACE-Stepに送信されたキャプション（**編集可能**） |
| **言語** | 使用された言語コード |
| **パラメータ** | 全パラメータのJSON表示 |

### キャプションの手動編集

デバッグパネルのキャプション欄は `<textarea>` であり、**直接編集が可能**です。編集した内容は次回の生成時に反映されます。

これにより、AIが生成したキャプションを微調整して再生成する、というワークフローが実現できます。

**♻️ 再生成ボタン**: キャプション欄の横にある再生成ボタンで、現在の設定に基づいてキャプションをLLMで再生成できます。

## 10.14　モード排他制御

Easy Musicの4つのモード（通常 / カバー / リペイント / JUKEBOX）は**相互排他**です。

| 操作 | 結果 |
|------|------|
| カバーON → リペイントON | カバーが自動OFF |
| リペイントON → JUKEBOX ON | リペイントが自動OFF |
| JUKEBOX実行中 → カバーON | JUKEBOX停止 → カバーON |
| カバーON → カバーOFF | 通常モードに戻る |

通常モードに戻すには、アクティブなモードのボタンをもう一度クリックしてOFFにします。
